
1. Ограничения:
	Таблица сканируется только в диапазоне A1:D100. Если диапазон выходит за границы, то требуется внести корретировку в скрипт. Скрипт жестко привязан к очередности стобцов в таблице. При изменении очередности потребуется внести корректровку в скрипт
	Адрес на сайте ЦБ остается неизменным, равно как и формат выдачи данных
	В целях обеспечения целостности БД, данные в БД записываются только после того, как все поля в таблице будут заполнены. При удалении хотя бы одного значения данные из БД стираются. Первичный ключ - номер заказа. Заказы с одинаком номером существовать не могут, в БД записывается последний экземпляр заказа.

	
2. Ключи и пароли
	БД создана со следующими параметрами входа:
		database: test1
		user: test1admin
		password: test
	Telegram токены:
		API_ID: 17961338
		API_HASH: ef23998614e7ccb3219cd07b4a63111d
		BOT_TOKEN: 5586720101:AAHRXxwbImY5_C0KupMSGyqBpwriL3UPrPk

3. Порядок запуска:

	3.1 Качаем скрипты из репозитория командой git clone https://github.com/SergeyKhoteev/KanalTest
	3.2 Создаем БД с параметрами из п.2
	3.3 Подключаем Telegram бота TestPyKanal. На него будут приходить уведомления
	
	Открываем в терминале папку KanalTest

	3.1 Устанаваливаем библиотеки из папки KanalTest командой pip install -r requirements.txt
	
4. Демонстрация

	4.1 Заходим на страницу https://docs.google.com/spreadsheets/d/1OV88E63sw48H4ZzYCaJIag0dXggpCiGYThDDzNJ8QEs/edit#gid=0. Это исходная таблица, откуда берутся данные
	4.2 Запускаем скрипт из папки командой python3 ./quickstart.py
	4.3 Смотрим Тг бота, туда будут приходить уведомления
	3.3 В соседнем терминале заходим в папку project (подпапка KanalTest) командой cd ./project/
	3.4 Запускаем сервер Django командой python3 ./manage.py runserver 7070 (Если этот порт 7070 занят, то выбирайте любой другой, открытый для пользователя)
	4.2 Данные с БД доступны по ссылке http://127.0.0.1:7070/
